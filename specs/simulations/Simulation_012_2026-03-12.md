## Semaine 1–4 : Sprint 1 — mode dégradé (github_repo absent)

Sprint 1 démarre sans github_repo dans projects.json. CI 1 (feature→develop) et CI 2 (develop→main) skippés (Y5). CI local R-6 (Albert QA & DevOps) vert → H4 (Sprint Review) déclenché (mode dégradé). R-7 (Nghia Stakeholder) approuve. Phase 1 sprint_complete : sprint_summary écrit. Phase 2 (merge_to_main) skippée car mode dégradé (Y5). ✅

### AA1 — Z1 mode dégradé : sprint_number non incrémenté après phase 1

La correction Z1 dit : sprint_number est incrémenté **seulement dans la phase 2 (merge_to_main)**, après CI 2 (develop→main) vert. Mais en mode dégradé (Y5), merge_to_main est skippée. Personne n'incrémente sprint_number.

Résultat : sprint_number reste à 1 après le sprint 1. Si l'opérateur invoque le sprint 2 avec `--start-phase E3 (Sprint Backlog) --thread-id project-sprint-02` : load_context charge sprint_number=1 depuis BaseStore (mémoire long terme) (incorrect). thread_id "sprint-02" ne correspond pas à sprint_number=1.

**Correction** : En mode dégradé (github_repo absent), la phase 1 (sprint_summary) incrémente directement sprint_number. Règle : sprint_number est incrémenté dans la phase de clôture applicable : phase 2 si CI 2 (develop→main) disponible, phase 1 si mode dégradé.

## Semaine 6 : Branch protection, merge_to_main

### AA2 — H4 (Sprint Review)' notation incohérente avec la convention H

La section III.17 (Z2) introduit « interrupt H4 (Sprint Review)' (`reason="pr_review_required"`) ». La notation H4 (Sprint Review)' ne correspond à aucun type d'interrupt dans le tableau III.6 (H1 (validation Gros Ticket)–H6 (résolution conflit Git)). La convention établie en V1 (III.13) est d'utiliser H5 (approbation escalade API payante) avec un champ `reason` pour distinguer les cas. "H4 (Sprint Review)'" crée une notation hors-spec.

**Correction** : Remplacer toute référence à "H4 (Sprint Review)'" par "H5 (approbation escalade API payante) avec `reason='pr_review_required'"`. Aucun nouveau type d'interrupt. Le script handle_interrupt.py affiche le message contextuel adapté à ce reason.
